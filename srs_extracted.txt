Introduction
Purpose
This Software Requirements Specification (SRS) document describes the functional and nonfunctional requirements of the Cultural Etiquette Advisory System (CEAS) Version 1.0.
The system provides culture-specific etiquette advice for different countries and regions based on selected scenarios such as greeting, dining, business meetings, and Gift Giving.
This document covers the complete scope of the advisory system including user access, rule-based advice generation, risk meter calculation, safe alternatives, and optional feedback and admin management.
Document Conventions
“Shall” indicates a mandatory requirement.
“Should” indicates a recommended feature.
“May” indicates an optional feature.
Requirements are numbered in the format: REQ-CEAS-xx
Priorities are mentioned as High / Medium / Low

Project Scope
The Cultural Etiquette Advisory System aims to reduce cultural misunderstandings by giving users accurate, scenario-based etiquette recommendations. The system provides:
Do’s and Don’ts
Severity levels
Explanations (why the advice matters)
Cultural Risk Meter
Safe Alternatives
This system supports respectful interaction and improves user confidence in multicultural environments.

Overall Description
Product Perspective
The Cultural Etiquette Advisory System is a new, standalone advisory software product. It works as a rule-based recommendation system where etiquette rules are stored in a structured knowledge base.The system retrieves cultural profiles and scenario-specific rules and displays a final advisory output to the user.
High-level flow: User → Select Country & Scenario → Fetch Culture Profile → Apply Etiquette Rules → Generate Advice → Show Risk Meter → Suggest Safe Alternative → Display Output
Product Features
Major features include:
User login/register or guest access
Country/Region selection
Scenario selection (Greeting, Dining, Business, Gift giving etc.)
Etiquette advice generation (Do’s & Don’ts + Explanation)
Severity and Cultural Risk Meter display
Safe alternatives suggestion
Save/export advice
Feedback collection
Admin module for managing cultural profiles and rules
User Classes and Characteristics
Guest User
Uses the system without registration
Can view etiquette advice only
Registered User
Can login and view advice
Can save/export advice
Can give feedback
Admin User
Has full privileges
Can add/edit/delete cultures, scenarios, and rules
Can review user feedback
Operating Environment
Platform: Web application / Desktop application (based on implementation choice)
Operating System: Windows 10/11, Linux, macOS
Browser (if web app): Chrome / Edge / Firefox
Language: Python / Java
Database: SQLite / MySQL / JSON dataset
Design and Implementation Constraints
The system must follow Object-Oriented Design principles
Knowledge base rules must be structured and maintainable
The project must be implementable within semester time
User-friendly interface is required
Limited dataset may be used due to project size constraints
The system should not depend on continuous internet (if rules are offline)
User Documentation
The following documents will be delivered:
User Guide (how to use system)
Admin Guide (how to manage rules)
README (installation steps)
Online help messages (optional)
Assumptions and Dependencies
Assumptions
Users provide correct country and scenario input
Rule dataset is accurate and validated
Dependencies
Depends on stored knowledge base (DB/JSON)
If export feature is included, depends on file generation library
If web app is used, depends on browser compatibility
System Features
User Authentication & Access
3.1.1 Description and Priority
The system shall allow users to access via login/register or guest mode. Priority: High
3.1.2 Stimulus/Response Sequences
User selects Login/Register
System validates credentials
System grants access or shows error
3.1.3 Functional Requirements
REQ-CEAS-01: The system shall allow user registration with name, email, and password.
REQ-CEAS-02: The system shall allow user login using valid credentials.
REQ-CEAS-03: The system shall provide guest access for viewing etiquette advice.
REQ-CEAS-04: The system shall display error message for invalid login.
Country/Region and Scenario Selection
3.2.1 Description and Priority
User shall select country/region and scenario to obtain advice. Priority: High
3.2.2 Stimulus/Response Sequences
User selects country
User selects scenario
System confirms selections
3.2.3 Functional Requirements
REQ-CEAS-05: The system shall display a list of available countries/regions.
REQ-CEAS-06: The system shall display available scenarios (Greeting, Dining, Business, Gift Giving).
REQ-CEAS-07: The system shall validate selections before generating advice.
Etiquette Advice Generation
3.3.1 Description and Priority
System shall generate etiquette advice using rules for selected culture and scenario.Priority: High
3.3.2 Stimulus/Response Sequences
User clicks “Get Advice”
System fetches rules from knowledge base
System generates advice
3.3.3 Functional Requirements
REQ-CEAS-08: The system shall retrieve cultural profile data based on selected country/region.
REQ-CEAS-09: The system shall fetch scenario-specific etiquette rules.
REQ-CEAS-10: The system shall display Do’s and Don’ts.
REQ-CEAS-11: The system shall display explanation/reason for each advice.
REQ-CEAS-12: The system shall display “Rules not available” message if data missing.
3.4 Cultural Risk Meter
3.4.1 Description and Priority
System shall show risk level based on severity of etiquette rules.Priority: Medium
3.4.2 Stimulus/Response Sequences
After advice generation
System checks rule severity
System shows risk level (Low/Medium/High)
3.4.3 Functional Requirements
REQ-CEAS-13: The system shall classify rules into severity levels (Important/Advisory/Optional).
REQ-CEAS-14: The system shall calculate risk meter value from severity.
REQ-CEAS-15: The system shall display risk meter to the user.
3.5 Safe Alternatives Recommendation
3.5.1 Description and Priority
System shall provide neutral fallback actions to reduce risk of cultural mistakes.Priority: Medium
3.5.2 Stimulus/Response Sequences
After advice generation
System identifies sensitive rules
System suggests safe alternative behavior
3.5.3 Functional Requirements
REQ-CEAS-16: The system shall generate safe alternative suggestions for each scenario.
REQ-CEAS-17: The system shall display safe alternatives along with the advice
3.6 Save/Export Advice
3.6.1 Description and Priority
System should allow users to save or export advice.Priority: Low/Optional
3.6.2 Stimulus/Response Sequences
User clicks save/export
System stores or downloads output
3.6.3 Functional Requirements
REQ-CEAS-18: The system should allow saving advice to user profile.
REQ-CEAS-19: The system may allow exporting advice in PDF/Text format.
3.7 Feedback Module
3.7.1 Description and Priority
User can submit feedback to improve system.Priority: Low
3.7.2 Functional Requirements
REQ-CEAS-20: The system may allow user ratings (1–5).
REQ-CEAS-21: The system may allow users to provide suggestions.
3.8 Admin Management Module
3.8.1 Description and Priority
Admin shall manage cultural profiles, scenarios, and rules.Priority: High
3.8.2 Functional Requirements
REQ-CEAS-22: The system shall allow admin to add/update/delete culture profiles.
REQ-CEAS-23: The system shall allow admin to add/update/delete scenarios.
REQ-CEAS-24: The system shall allow admin to add/update/delete etiquette rules.
External Interface Requirements
User Interfaces
Login/Register page
Country and Scenario selection page
Advice output page (Do/Don’t + explanation + risk meter + alternatives)
Admin dashboard (manage rules)
Feedback form
Hardware Interfaces
No special hardware interfaces are required.
System runs on normal computer/mobile device.
Software Interfaces
Database: SQLite / MySQL / JSON
Backend: Python/Java
UI Framework: Streamlit / Flask / Java Swing (based on implementation)
Communications Interfaces
If web-based: uses HTTP/HTTPS
If local app: no external communication required
Other Nonfunctional Requirements
Performance Requirements
Advice generation should be completed within 2 seconds for normal dataset size.
The system shall support at least 50+ rules without lag.
Safety Requirements
System shall display etiquette advice as guidance only.
System shall not encourage offensive or harmful actions.
Security Requirements
Login passwords shall be stored securely (hashed if DB used).
Only admin can modify cultural rules and profiles.
User data shall not be accessible to other users.
Software Quality Attributes
Usability: Easy navigation and understandable advice
Maintainability: Modular code structure (OOSE)
Reliability: Consistent advice generation
Scalability: Easy to add new countries and scenarios
Portability: Works on multiple OS platforms
Other Requirements
Knowledge base should store:
Country/Region
Scenario
Do/Don’t
Explanation
Severity level
Optional multilingual support may be added in future.
Appendix A: Glossary
Etiquette: Socially acceptable behavior in a culture
Scenario: Situation like dining, greeting, business meeting
Rule Engine: Logic system applying cultural rules
Risk Meter: Displays the risk level of cultural mistakes
Safe Alternative: Neutral respectful fallback action
